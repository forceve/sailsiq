# SailSIQ 平台（数字产品线）MVP 设计总纲

> 核心目标：做出好用的复盘/教学工具，用最小代价验证 **“有人愿意用 + 愿意配硬件”**。

---

## 0. 一句话定位（Positioning）
- SailSIQ 不单是轨迹播放器，而是集教学/复盘/分享功能于一身的超级工具。支持多种数据导入，支持尽可能多的帆船场景。

### 0.1 服务形态调整（Local-first Service Strategy）

- **默认服务形态从“基础云服务”调整为“本地工作目录优先，云端能力可选”**
- 原因 1：复盘/讲解场景经常发生在网络不稳定环境，依赖云端会直接影响可用性
- 原因 2：视频文件、原始日志、硬件导出文件体量大，强依赖上传会拉低导入、打开、对齐与讲解体验
- 因此 MVP 的核心不是“把 Session 存到云上”，而是“围绕一个本地工作目录组织 Session、视频、导出物与讲解素材”
- 云端的角色后移为：可选同步、远程备份、分享发布、团队协作与数据收集
- 产品原则调整为：
  - Offline-first：断网可打开、可回放、可标注、可讲解
  - Local-first：数据、视频、导出物默认保存在本地工作目录
  - Cloud-optional：只有在用户明确需要分享、协作、备份时才进入云流程
  - Progressive enhancement：浏览器版优先通过本地目录能力落地；不支持相关 API 的环境提供降级路径

### 0.2 思考过程摘要（Why Service Modes Split）

- 本次调整不是简单地“反对云服务”，而是重新识别不同用户群体的真实工作方式
- 云端服务更适合两类场景：
  - **机构管理场景**：俱乐部、学校、训练营、青少年培训体系，需要统一管理资料、人员、训练计划与复盘记录
  - **强网络、强组织关联场景**：教练、队员、管理者长期处于同一组织结构中，需要跨人协作、权限管理、集中沉淀
- 本地工作目录更适合两类场景：
  - **赛队使用场景**：现场复盘、讲解、视频对齐、素材整理更强调速度、稳定性与大文件处理能力
  - **个人使用场景**：个人训练、个人分析、个人素材管理，不希望先上传再使用
- 一个需要特别保留云服务的边缘场景是：
  - **业余赛队异地协作**：虽然不是机构化训练体系，但队员和教练未必经常聚在一起，因此仍然需要通过云端分享 Session、讲解材料与导出成果
- 因此最终结论不是“二选一”，而是：
  - 本地工作目录服务赛队和个人的主工作流
  - 云服务服务机构管理、青少年训练和异地协作分享
  - 产品架构需要同时支持这两种模式，但默认入口应当以本地工作目录为先

## 1. 用户与场景（Users & Use cases）

### 1.1 目标用户
- 教练（Coach）：需要快速产出讲课素材、复盘内容、可分享链接
- 竞赛/训练队员（Sailor）：想知道自己哪里慢/怎么改
- 俱乐部/训练营（Club/Camp）：想沉淀课程、统一训练流程
- 内容创作者（Content Creator）：需要将航行数据可视化叠加到视频中，制作教学/赛事/体验类内容

### 1.2 MVP 聚焦场景（必须能打通）
- 赛后复盘：导入一次航行到本地工作目录 → 回放 → 标注关键节点
- 课堂/讲解：地图+航迹+航标+风箭头 + 本地视频 → 跟随时间讲述战术

> P0 先收口在“能复盘、能教学、能离线保存到本地工作目录”；P1 以“视频辅助模式”和“本地素材组织能力”作为最高优先级，其他增强项再按 P1 / P2 递进推进。

> 非 MVP 场景：高精度帆船动力学、全自动战术结论、复杂风场连续插值

### 1.3 服务形态与用户分层

- **本地工作目录优先用户**
  - 赛队
  - 个人 sailor
  - 现场复盘 / 线下讲解 / 大视频素材处理用户
- **云服务优先用户**
  - 俱乐部
  - 青少年训练机构
  - 有明确组织架构、权限管理和资料沉淀需求的团队
- **混合模式用户**
  - 业余赛队
  - 平时分散、比赛或训练时短暂聚集的团队
  - 这类用户的主编辑过程应当仍在本地完成，但需要方便地发布到云端用于异地共享

## 2. 功能拆解与优先级（P0 / P1 / P2）

### 2.1 P0（MVP 必须有：能复盘、能教学）

#### A) 工作目录与 Session 体系（Workspace / Session）
- Workspace（本地工作目录）是 MVP 的一级对象，用于承载多个 Session、视频、导出物、截图与课程素材
- Session（一次航行）是 Workspace 内的核心业务对象
- Session 基本信息：名称、日期、地点（可自动填）、船型（可选）
- Session 与其轨迹、标注、航标、视频引用、导出物默认都保存在同一 Workspace 内
- Project 不再作为 P0 必需抽象；若保留，也只是 Workspace 内的可选分组字段

#### B) 本地工作目录（Local Workspace）
- 允许用户：
  - 创建一个新的本地工作目录
  - 打开一个已有的本地工作目录
  - 在工作目录中浏览、搜索、打开 Session
- 工作目录需要有清晰的目录结构与元数据文件，保证后续可迁移、可备份、可打包、可同步
- 默认保存策略：用户的导入结果、标注、航标、截图、视频引用都直接落盘到工作目录，而不是只停留在浏览器内存
- Workspace 目录设计必须足够清晰，让用户知道“轨迹放哪里、视频放哪里、Session 归档放哪里”，并且系统可以自动扫描识别

#### C) 导入（Import）
- 支持格式：
  - 普通 GPX 文件：导入 .gpx
  - SailSIQ 设备 bin：导入设备导出的 .bin（二进制文件，可能包含 heading 等额外字段）
- 解析最小字段：`time, lat, lon`（可推导 speed/heading）
- **SailSIQ 设备 bin 可能包含大部分信息（如 heading、速度、航标位置、风向等），系统会自动识别并填入，减少手动输入**
- 允许缺失字段：UI 清晰显示 N/A，不阻塞使用
- 导入后的源文件可选择：
  - 引用原路径（适用于用户自己管理目录）
  - 复制/归档到当前 Workspace（推荐，便于后续迁移与分享）
- 新建 Session 时支持三种方式：
  - 选择文件并复制到 Workspace
  - 选择本机已有文件并直接绑定路径
  - 从 Workspace 已自动识别出的轨迹文件中直接创建

#### D) 地图画板 v1（Canvas v1）
- 底图：矢量/卫星切换
- 航迹：轨迹线（polyline），支持渐进绘制
- 航标/航点 Marks：添加/拖拽/命名（Start/Mark/Finish）
- 风（最简）：全局风向/风速（手动输入即可）

#### E) 回放（Replay）
- 播放/暂停
- 倍速：1x–10x 滑块调节（颗粒度 0.1x）
- 时间轴进度条：拖动跳转到任意时间点
- 时间跳转：±10秒 / ±30秒 快捷按钮
- 回放同步：船位点移动、轨迹推进

#### F) 图表与统计 v1（Charts & Stats）
- 最少 2–3 个图：
  - Speed（SOG 估算/设备提供）
  - Heading/Course（相邻点方位角）
  - Turn rate（可选）
- 关键统计卡片：
  - Duration / Distance / Avg speed / Max speed
  - Turn count（粗略：转向次数）
- 图表联动回放：点击曲线 → 时间轴跳转

#### G) 标注（Annotations）教学核心
- 时间轴事件点 Event marker：文本备注
- 事件列表：可跳转定位

#### H) 本地持久化与恢复
- 用户关闭页面后再次打开，能够从同一 Workspace 恢复 Session 列表与最新编辑结果
- 标注、航标、视频 offset、导出记录等修改应即时保存或显式保存到本地
- 系统打开 Workspace 时应自动扫描标准目录，识别：
  - 待创建 Session 的轨迹文件
  - 可绑定的视频文件
  - 已失效的外部路径引用
- 在无法使用本地目录 API 的环境下，至少提供：
  - 浏览器本地缓存/OPFS 降级
  - 导入/导出 Workspace 包（zip）作为备选

### 2.2 P1（优先做“更强的讲解与表达”，并补齐本地素材组织）

> P1 的第一优先级：视频辅助模式 + Workspace 内的视频/素材管理。先把“视频+数据同步讲解”做出来，再补齐教学工作台、导出分享和课程生产能力。

#### I) 视频辅助模式（Video Overlay Mode）- P1 第一优先级

- 多视频类型支持：

    - 支持导入 普通平面视频 与 360° 全景视频（如运动相机/头盔相机）

    - 视频与 Session 时间轴对齐（手动或自动校准）
    - 支持本地文件视频作为主路径，远端 URL 视频作为补充路径

- 三种显示模式（可切换）：

    1. 视频叠加模式（Overlay）：在视频画面上叠加高对比度数据仪表（速度、航向、时间、关键指标等），适合沉浸式观看与讲解动作细节。

    2. 分屏对照模式（Split View）：页面左右或上下分屏：一侧为视频，另一侧为数据与画板，两边时间对齐同步操作。数据侧功能与常规 Canvas 一致（航迹、航标、风、时间轴），适合战术讲解与视频动作的同步对照。

    3. 视频辅助模式（Dual Window Video Companion）：在现有 Replay Workspace 之外额外打开一个独立的视频辅助窗，主窗口继续承载地图、时间轴、图表与播放控制，辅助窗专注视频展示与必要 HUD。两个窗口绑定同一 Session，并保持时间、播放/暂停、倍速、视频 offset 与视频源切换同步；在 `Sync` 校准过程中，辅助窗需实时跟随主窗口的校准预览帧变化。该模式默认隐藏与数据工作流无关的侧边栏、图表区和重复控制区，适合双屏讲解、投屏、小屏设备外接显示等场景。

- 360° 视频交互：支持视角拖动

#### J) Workspace 素材管理

- 在 Workspace 内管理：
  - 视频文件
  - 截图
  - 导出 PDF / 视频组件
  - Lesson Capsule 素材
- 支持显示素材是否“已归档到当前 Workspace”还是“外部路径引用”
- 支持素材丢失检测与重新定位（例如视频文件被移动后重新绑定）
- 视频绑定支持：
  - 复制到 Workspace
  - 引用 Workspace 内相对路径
  - 直接引用本机外部路径

#### K) 教学工作台补齐

- 风箭头显示
- 演示模式 / 全屏讲解模式
- 更快的事件标注交互（替代 prompt 式输入）
- 一键截图 Snapshot

#### L) Snapshot / Export / Share

- 一键截图 Snapshot（保存画板当前状态）
- 分享链接（只读 view，可基于本地打包后上传或选择已有云空间发布）
- 导出 PDF（最简版即可）
- 视频仪表盘导出：可选组件分别导出透明背景视频（MP4），包括轨迹、速度、航向三个独立组件
- Workspace 导出：支持将单个 Session 或整个 Workspace 打包，便于转移机器与离线分发

#### M) A/B 对比（Compare）

- 同一 Session：两条片段对比
- 或两个 Session：轨迹叠加对比

#### N) Lesson Capsule（课程胶囊）

- 定位：解决教练 Briefing 素材的痛点，提高教学质量，减少教练无用工作量
- 从事件点选 3–8 个
- 自动生成可滑动故事流：截图 + 关键数据 + 解释文本（原因/Comment）

#### O) 轨迹清洗与性能

- 去抖/滤波：速度曲线与位置抖动平滑（简单移动平均或更稳一点的滤波）
- 去掉静止段（港池/停船）：可设置阈值（speed < x 持续 y 秒）
- 轨迹抽稀（简化点数）：保证地图渲染与回放不卡
- 轨迹分段（upwind/downwind）

### 2.3 P2（技术壁垒方向：做强就很难抄）
> P2 的目标：把 SailSIQ 从“轨迹/数据播放器”升级成教练可信赖的技术/战术解释系统，并在此基础上提供可选云协作能力。

这些能力往往需要长期数据积累与工程打磨，短期难以被复制。

#### P) Session 组织与团队能力

- Project 体系：用于组织多个 Session
- 队伍 / 团队字段与管理

#### Q) 自动标红（Auto highlights v0.1）

- **路径速度渲染（Speed gradient track）**：
  - 轨迹按 `speed` 映射为渐变色（低速→高速），回放时同步“走色/渐进绘制”
  - 可选：在图例中显示 min/avg/max speed（或分位数）
- **疑似问题点标注（Timeline markers）**：
  - 系统自动生成“疑似问题点” marker 并标在时间轴上（可过滤/可分类渲染）
  - 点击 marker：时间轴跳转 + 地图定位到对应点
  - MVP 级检测规则（启发式即可）：
    - 速度骤降（speed drop）：短窗口内 speed 明显下滑（相对阈值/分位数）
    - 转向事件（turn event）：Exit 角度过大 / Turn rate 峰值明显
- **输出原则：只是识别几种简单异常，方便快速定位**：
  - 教练可对 marker：确认/否决 + 选择原因（rudder/timing/wind/route/other）+ 添加 Comment（可选）
  - 系统保存修正，用于后续提升规则/模型（形成标签飞轮）

#### R) 可选云同步与发布（Optional Cloud Sync / Publish）

- 将本地 Workspace 或其中的 Session 同步到云端
- 提供远程备份、多设备拉取、只读分享发布、团队协作入口
- 同步对象优先级：
  - 元数据、标注、航标、导出记录优先
  - 视频与原始日志可按需选择“仅引用 / 压缩上传 / 不同步”
- 云不是系统唯一真相源；本地 Workspace 才是用户主工作副本

#### S) 数据收集与隐私

- 默认私有 + opt-in
- 数据使用开关（Raw/Features/Labels）
- 数据导出、删除

- 会员功能：提供完备的云服务
- 硬件日志深度接入（与 SailSiq 硬件结合打造更好的品牌生态）
- 战术语义引擎（自动切片/事件识别/事件归因/个性化）
- 风场重建（根据风力仪 + 航行数据 + POLARS 重建风场）
- 团队云空间、协作、权限、评论、作业式教学（按成本提供免费版本）
    - Team workspace：教练/队员/管理员权限（查看/评论/编辑/导出/分享），团队工作流整合（训练计划/个人档案/智能分析），云端服务
    - 评论与讨论：在事件点上评论、@队员、形成复盘记录/训练计划
    - 作业式训练：教练布置“看这 5 个事件并回答问题/打标签”，系统收集反馈作为训练数据
    - 内容库：Lesson Capsule 与 Session 版本管理，形成训练营知识沉淀与复用

## 3. MVP 信息架构（App IA）
### 3.1 页面清单（Pages）

1. Workspace Home / Sessions 列表
2. Workspace Setup（创建 / 打开本地工作目录）
3. New Session（导入数据 / 画板创建）
4. Replay Workspace（核心工作台）

   * Map Canvas
   * Timeline
   * Charts
   * Events panel
5. Export / Share / Publish
6. Settings（含 Data & Privacy）

### 3.2 核心工作台布局（Workspace layout）

* 左：事件/航标列表（Events & Marks）
* 中：画板地图（Canvas）
* 右：曲线/统计（Charts & Stats）
* 底：时间轴（Timeline）

（桌面/浏览器优先，多面板体验更爽；移动端再做折叠。）

### 3.3 页面设计概要（Page Design Brief）
- **Workspace Home / Sessions 列表**
  - 顶部状态：当前 Workspace 名称、路径、是否可写、最近同步状态（如有）。
  - 列表：Session 卡片（名称/日期/来源/标记数/素材状态），支持搜索/筛选。
  - 快捷入口：新建 Session（导入 / 画板）、打开工作目录、最近打开、示例数据。
  - 状态：空列表占位（引导创建或打开 Workspace）、加载中骨架屏、错误提示。
- **Workspace Setup（创建 / 打开本地工作目录）**
  - 创建模式：创建新 Workspace、打开已有 Workspace、导入 Workspace 包。
  - 权限状态：目录授权状态、浏览器兼容性、降级方案提示。
  - 结构提示：说明 Session、视频、导出物会如何组织到目录中。
- **New Session（创建/加载 GPX）**
  - 信息区：名称、日期、地点自动填、船型。
  - 数据导入区：GPX / SailSIQ bin 上传卡片，显示解析预览与字段健康度。
  - 保存策略：明确选择“引用原文件”或“复制到当前 Workspace”。
  - 画板模式入口：明确的“改用手动绘制”切换。
  - 辅助提示：缺失字段提示、不阻塞继续；导入成功后展示轨迹缩略图。
- **Replay Workspace（核心工作台）**
  - 左：事件列表 / 航标列表切页，支持跳转定位与过滤。
  - 中：地图画板，轨迹渐进绘制，航标，选中点高亮。
  - 右：图表与统计卡片，曲线联动时间轴；指标卡可折叠。
  - 底：时间轴 + 播放控制（播放/暂停、倍速、±10s/±30s、拖拽跳转）。
  - 工具条：一键标注入口、素材状态、保存状态、视频绑定状态。
- **Export / Share / Publish（P1/P2）**
  - 导出选项：PDF（轨迹+事件+截图）、Workspace 包、视频仪表组件导出。
  - 发布选项：分享链接（只读视图）、可选云同步/发布。
  - 预览区：PDF/分享链接预览，选择包含的事件与截图。
  - 状态：导出中进度/失败重试；链接权限提示；本地包与云发布状态分离。
- **Settings（含 Data & Privacy）**
  - 数据与隐私：数据收集开关、匿名化说明、条款链接。
  - 设备与导入：默认导入偏好（速度单位、时间格式）、设备固件版本显示、默认归档策略。
  - 工作目录：默认 Workspace、目录权限、兼容性检测、缓存清理。
  - 账户：语言、主题、云端登录占位（P2 预留）。

## 4. MVP 交互流程（User Flows）

### P0 基础流程（MVP 必须）

#### Flow A：创建 / 打开本地工作目录

1. **进入 Workspace Setup**
   - 首次进入应用时，优先引导用户创建或打开本地工作目录
   - 用户可选择：
     - **选项1：创建新的 Workspace**
     - **选项2：打开已有 Workspace**
     - **选项3：导入 Workspace 包**（降级或迁移路径）
   - 系统完成目录权限请求与能力检测

2. **进入 Workspace Home**
   - 显示当前 Workspace 名称、路径、Session 列表与最近素材
   - 从此处进入新建 Session、打开既有 Session、导出或发布

#### Flow B：赛后复盘（导入数据路径）

3. **创建 Session - 选择数据源**
   - 从 Workspace Home 页面点击"新建 Session"
   - 选择创建方式：
     - **选项1：导入数据**（GPX 文件或 SailSIQ 设备 bin）
     - **选项2：画板模式**（手动绘制，见 Flow C）
   - 填写基本信息：名称、日期、地点（可自动识别）、船型（可选）

4. **导入数据**（选择导入数据时）
   - 导入 GPX 文件或 SailSIQ 设备 bin
   - 系统自动解析：`time, lat, lon`（派生 speed/heading）
   - **SailSIQ 设备 bin 可能包含大部分信息（如 heading、速度、航标位置、风向等），系统会自动识别并填入**
   - 缺失字段显示 N/A，不阻塞使用
   - 自动生成轨迹预览
   - 选择保存策略：
     - 引用原文件
     - 复制到当前 Workspace（推荐）

5. **补充航标与风信息**（如导入数据中缺失）
   - 在地图上添加/拖拽航标：Start、Marks、Finish
   - 手动输入全局风向/风速（如未自动识别）

6. **回放与观察**
   - 播放/暂停控制
   - 倍速调节：1x–10x 滑块（颗粒度 0.1x）
   - 时间轴拖动跳转到任意时间点
   - 快捷按钮：±10秒 / ±30秒
   - 观察船位点移动、轨迹渐进绘制
   - 查看右侧图表：Speed、Heading/Course、Turn rate
   - 查看统计卡片：Duration、Distance、Avg/Max speed、Turn count
   - 点击图表曲线 → 时间轴自动跳转

7. **标注关键事件**
   - 在关键时刻点击"+事件"
   - 输入文本备注（一句话描述）
   - 事件自动添加到左侧事件列表
   - 从事件列表可快速跳转定位

8. **保存到 Workspace**
   - 保存为 Session（可用于后续讲课）
   - 写入 Session 元数据、轨迹、标注、航标与素材引用
   - 更新 Workspace 索引

9. **离线再次打开**
   - 用户断网重新进入应用
   - 打开同一个 Workspace，继续回放、标注与讲解

10. **导出与分享**（纳入 P1/P2）
   - 导出 PDF（包含轨迹、事件、截图）
   - 导出 Workspace 包或 Session 包
   - 生成分享链接（只读视图，需进入发布流程）

#### Flow C：画板模式（手动绘制路径）

11. **创建 Session - 选择画板模式**
   - 从 Workspace Home 页面点击"新建 Session"
   - 填写基本信息：名称、日期、地点、船型（可选）
   - **选择创建方式**：画板模式

12. **手动绘制轨迹**
   - 在地图上手动绘制航迹（点击/拖拽绘制路径）
   - 添加/拖拽航标：Start、Marks、Finish
   - 手动输入全局风向/风速

13. **保存为 Session**
    - 保存为 Session（可用于后续讲课）
    - 结果直接写入当前 Workspace

#### Flow D：制作/执行课程（教练）

14. **打开 Session**
    - 从 Workspace Home / Sessions 列表选择已有 Session
    - Session 来源可以是：
      - Flow B（导入数据）创建的 Session
      - Flow C（画板模式）创建的 Session

15. **准备讲解内容**
    - 查看事件列表（手动标注的事件，如有）
    - 手动选择要讲解的事件点顺序
    - 或按时间顺序自动排列

16. **在当前工作台中讲解**
    - 使用地图画板 + 时间轴 + 关键数据仪表进行讲解
    - 跟随时间轴讲述战术
    - 可随时跳转到特定事件点

### P1 增强流程

#### Flow A 增强：视频辅助模式（最高优先级）

17. **导入与对齐视频**
    - 导入普通平面视频或 360° 全景视频（如运动相机/头盔相机）
    - 优先使用本地文件视频，并支持归档到 Workspace
    - 视频与 Session 时间轴对齐（手动或自动校准）

18. **三种显示模式（可切换）**
    - **视频叠加模式（Overlay）**：在视频画面上叠加高对比度数据仪表（速度、航向、时间、关键指标等），适合沉浸式观看与讲解动作细节
    - **分屏对照模式（Split View）**：页面左右或上下分屏：一侧为视频，另一侧为数据与画板。数据侧显示效果与常规 Canvas 基本一致（航迹、航标、风、时间轴），适合战术讲解与视频动作的同步对照
    - **视频辅助模式（Dual Window Video Companion）**：打开独立视频辅助窗，主窗口保留完整 Replay 工作流与控制能力，辅助窗仅展示视频与必要 HUD，并与主窗口保持时间、播放状态、倍速、offset、视频源切换同步；在 `Sync` 校准时辅助窗实时跟随预览帧，适合双屏讲解、投屏和小屏扩展显示
    - 360° 视频交互：支持视角拖动

19. **视频模式讲解**
    - 如 Session 已关联视频，切换到视频叠加模式、分屏对照模式或视频辅助模式
    - 在视频画面上叠加数据仪表，讲解动作细节
    - 或使用分屏对照模式、视频辅助模式，同步讲解战术与动作

#### Flow B 增强：对比、导出与课程生产

20. **片段或 Session 对比**
    - 同一 Session：选择两条片段对比
    - 或两个 Session：轨迹叠加对比
    - 分屏显示，同步回放

21. **生成课程故事流**
    - 从事件点选择 3–8 个关键事件
    - 系统自动生成可滑动故事流：截图 + 关键数据 + 解释文本（原因/Comment）
    - 预览并编辑内容
    - 导出为可分享格式

22. **导出与分享**
    - 导出 PDF（包含轨迹、事件、截图）
    - 导出 Workspace 包 / Session 包
    - 生成分享链接（只读视图，需用户主动发布）

### P2 后置流程

#### Flow A 后置：自动标红与标签飞轮

23. **自动识别疑似问题点**
    - 轨迹按速度渲染渐变色（低速→高速），回放时同步"走色/渐进绘制"
    - 系统自动识别：速度骤降、转向事件（tack/gybe）
    - 时间轴上显示自动生成的 marker
    - 点击 marker：时间轴跳转 + 地图定位到对应点（可高亮 5–15 秒窗口）

24. **教练修正与标签**
    - 教练可对 marker：确认/否决 + 选择原因（rudder/timing/wind/route/other）+ 添加 Comment（可选）
    - 系统保存修正，用于后续提升规则/模型（形成标签飞轮）

#### Flow B 后置：云同步 / 发布 / 协作

25. **按需同步到云端**
    - 用户主动选择要同步的 Workspace / Session
    - 保存元数据、标注、航标、导出记录到云端
    - 视频和原始文件按策略选择：引用 / 上传 / 不同步

26. **发布与协作**
    - 生成只读分享链接或团队协作副本
    - 数据收集与模型训练必须保持 opt-in

## 5. 数据模型最小集合（MVP Schema）

### 5.1 本地工作目录建议结构

```text
workspace/
  workspace.json
  incoming/
    track/
    video/
  library/
    source/
    video/
  sessions/
    <sessionId>/
      session.json
      track.json
      events.json
      marks.json
      bindings.json
      assets/
        source/
        video/
        snapshots/
        exports/
  cache/
  index/
```

### 5.2 Workspace 元数据

- `workspace.json`
  - `id`
  - `name`
  - `version`
  - `createdAt`
  - `updatedAt`
  - `sessionsIndex`
  - `syncProfiles`（可选）

### 5.3 Session 最小模型

- `session.json`
  - `id`
  - `name`
  - `date`
  - `location`
  - `source`
  - `boatType`
  - `teamName`
  - `projectId`
  - `storageMode`：`workspace_copy | workspace_reference`
  - `stats`
  - `assetRefs`
  - `createdAt`
  - `updatedAt`

### 5.4 Asset 引用原则

- 所有大文件都不直接塞进 Session 主 JSON
- 对视频、原始导入文件、截图、导出物统一使用 `assetRefs`
- `assetRefs` 记录：
  - `kind`
  - `path`
  - `origin`
  - `status`
  - `size`
  - `hash`（可选）
- 优先使用相对路径，避免换机器后失效

### 5.5 云同步模型原则

- 云端副本不是唯一真相源
- 同步记录需要至少包含：
  - `workspaceId`
  - `sessionId`
  - `lastSyncAt`
  - `syncState`
  - `conflictState`
  - `remoteVersion`

### 5.6 浏览器技术路线

- 首选：File System Access API，直接绑定用户选择的本地工作目录
- 降级：OPFS / IndexedDB 缓存 + Workspace 包导入导出
- 后续如果需要更稳的本地目录能力，可再评估 Electron / Tauri 桌面壳

## 5.7 现有系统改造建议（面向当前代码）

### 5.7.1 当前系统现状

- 前端当前以 API 资源模型为中心：`Session / Track / Event / Mark` 都通过 `app/frtend/src/services/api.ts` 访问远端接口
- 后端当前是一个基于 Hono + KV 的轻量 API，承担导入解析、Session CRUD 与视频 URL 关联
- 视频功能已经具备一部分“本地优先”能力：`ReplayWorkspacePage` 支持选择本地视频文件，通过 `objectURL` 播放，并用 `localStorage` 保存部分偏好
- 这说明系统并不需要推倒重来；最合理的路径是把“本地文件视频”这套思路推广到整个 Workspace / Session 数据层

### 5.7.2 改造原则

- 不要直接把所有页面逻辑改成“直接读写文件”
- 先引入统一的存储抽象层，再把现有 API 适配为其中一种后端
- 让“本地工作目录存储”和“云端 API 存储”都实现同一套仓储接口

### 5.7.3 建议的改造顺序

1. 增加 `workspace repository` / `session repository` 抽象
   - 前端页面不再直接依赖 `sessionApi / trackApi / eventApi / markApi`
   - 页面依赖统一的 `repository` 接口，例如 `listSessions/getSession/saveSession/saveTrack/saveEvents`

2. 新增本地存储适配器
   - 使用 File System Access API 实现 `localWorkspaceRepository`
   - 负责目录初始化、文件读写、索引刷新、资源归档、相对路径管理

3. 保留当前后端，转为云适配器
   - 当前 `api.ts` 与 Hono/KV 后端不删除
   - 它们从“默认数据层”降级为“cloudRepository / publish adapter”

4. 调整 New Session 页面
   - 当前 `NewSessionPage` 在导入时直接调用 `/v1/parser/preview` 和 `/v1/sessions/import`
   - 改为：
     - 解析逻辑尽量前移到本地
     - 导入完成后写入当前 Workspace
     - 如用户选择云发布，再调用远端接口

5. 调整 Replay Workspace 页面
   - 当前 Replay 页对视频已有本地文件处理逻辑
   - 将同样的本地读写能力扩展到轨迹、事件、航标、截图、导出记录
   - 保存标注、航标、视频 offset 时优先写本地 Workspace

6. 增加 Workspace 上下文
   - 增加当前 Workspace 的路径、权限、读写状态、同步状态
   - Home 页面与 Settings 页面都需要展示 Workspace 状态

7. 最后再做云同步
   - 云同步不应该阻塞本地功能落地
   - 等本地结构稳定后，再设计 manifest、版本号、冲突合并和远程备份

### 5.7.4 受影响最大的现有模块

- `app/frtend/src/services/api.ts`
  - 需要从“页面直连 API”改成“提供云端适配器或基础 transport”
- `app/frtend/src/pages/NewSessionPage.tsx`
  - 需要增加 Workspace 选择、保存策略、导入后本地写入
- `app/frtend/src/pages/ReplayWorkspacePage.tsx`
  - 需要把当前本地视频能力扩展为本地 Session 资产能力
- `app/frtend/src/types/models.ts`
  - 需要补充 Workspace、AssetRef、storageMode、syncState 等模型
- `app/backend/src/index.tsx`
  - 保留为云同步 / 发布 / 备份服务，不再假设它是唯一数据源

### 5.7.5 风险与规避

- 浏览器兼容性风险
  - 通过 File System Access API + OPFS / zip 降级解决
- 目录权限失效风险
  - 需要权限状态检测、失效提醒、重新绑定流程
- 大文件管理风险
  - 默认归档为相对路径，避免复制过多；同时允许用户选择“复制到 Workspace”
- 云端与本地冲突风险
  - P0/P1 先避免双写；P2 再引入同步状态与冲突处理

## 12. 一句话 MVP 指导原则

> **MVP 不追求“自动教练”，只追求“让教练更快讲清楚”。**

做到了这点，你的硬件数据、语义引擎、风场重建都会自然接上。
