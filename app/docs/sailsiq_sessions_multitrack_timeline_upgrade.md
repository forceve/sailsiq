# SailSIQ Sessions 多轨时间轴升级方案（简版）

> 目标：为 `sessions` / Replay Workspace 增加“总时间轴 + 视频时间轴 + 工作区时间轴”的三轨结构，让用户能以“局部聚焦 + 全局参照”的方式完成视频对齐与分段复盘。

---

## 1. 设计目标

当前时间轴是单轨结构，适合基本播放与跳转，但不适合以下场景：

- 用户需要先看整场 Session 的全局位置，再聚焦其中一小段做反复复盘
- 用户需要把视频与整场 Session 的全局时间对齐，而不是只和局部片段对齐
- 用户需要在一个缩放后的工作区里操作播放、快进、倍速和事件分析

本次升级的核心目标是：

- 保留 Session 全局时间参照
- 允许用户从全局时间轴中裁出一个工作区
- 让视频始终与总时间轴对齐
- 让播放控制、事件、图表和数据分析只作用于工作区

---

## 2. 核心概念

### 2.1 总时间轴（Global Timeline）

- 表示完整 Session 的总时长
- 是所有时间坐标的全局基准
- 用于展示：
  - 当前全局播放位置
  - 工作区窗口所在位置
  - 视频片段在整场 Session 中的覆盖位置

### 2.2 视频时间轴（Video Timeline）

- 与总时间轴使用同一时间坐标系
- 视频条的起点为 `videoOffsetMs`
- 视频条的终点为 `videoOffsetMs + videoDurationMs`
- 用户拖动视频条，本质上是在修改视频与 Session 的对齐偏移 `offsetMs`

说明：

- 视频不与工作区对齐
- 视频只与总时间轴对齐
- 工作区只是观察和操作的局部窗口，不改变视频的全局定位关系

### 2.3 工作区时间轴（Workspace Timeline）

- 是从总时间轴中裁切出来的一段时间窗口
- 范围为 `workspaceStartMs ~ workspaceEndMs`
- 工作区条本身的完整长度只代表这一段局部内容
- 用户对播放、拖拽、快进、倍速的所有操作，都作用在工作区时间轴上

说明：

- 工作区有自己的局部视觉长度
- 但工作区的位置始终绑定在总时间轴中的某一段
- 当前播放时间仍然是“全局时间”，只是播放范围被限制在工作区内

---

## 3. 三轨关系

建议的垂直排列顺序：

1. 工作区时间轴
2. 总时间轴
3. 视频时间轴

其中：

- 工作区时间轴负责“操作”
- 总时间轴负责“定位”
- 视频时间轴负责“对齐”

这是一个典型的 `focus + context` 结构：

- `context`：总时间轴，告诉用户当前正在整场 Session 的哪一段
- `focus`：工作区时间轴，提供更高精度的局部回放与分析

---

## 4. 交互原则

### 4.1 工作区裁切

用户在总时间轴上操作一个 `workspace window`：

- 拖动左右 handle：改变工作区起止范围
- 拖动中间区域：整体平移工作区
- 点击总时间轴空白区域：快速跳转并重定位工作区

### 4.2 视频对齐

用户在视频时间轴上拖动整条视频片段：

- 向左拖：视频在全局时间中提前
- 向右拖：视频在全局时间中延后
- 拖动完成后更新 `offsetMs`

初版建议：

- 视频条长度由视频时长决定，不允许手动改长度
- 初版只支持整体平移，不支持 trim

### 4.3 播放与控制

播放、暂停、拖拽播放头、快进、倍速，只绑定工作区时间轴：

- 播放只在工作区内运行
- 到达工作区末尾时停止
- 快进和回退也在工作区范围内 clamp
- 工作区内的播放头同时映射到总时间轴和视频时间轴

### 4.4 数据显示范围

所有数据密集型内容只跟工作区绑定：

- 事件 marker
- 图表曲线
- 事件列表高亮
- 地图高亮区间
- 分析统计的局部上下文

工作区之外的细节内容不显示，只保留总时间轴上的全局参照。

---

## 5. 为什么不覆盖总时间轴

推荐方案：**总时间轴保留，细节只显示在工作区内。**

原因：

- 视频需要和总时间轴共用同一坐标系才能完成直观对齐
- 用户需要知道当前工作区位于整场 Session 的哪个位置
- 用户的大部分复盘行为是“按段聚焦”，而不是“整场粗看”
- 如果把细节直接铺满总时间轴，长 Session 下可读性会迅速下降

因此更合理的设计是：

- 总时间轴保留全局信息，但尽量稀疏
- 工作区承载高密度操作与数据
- 视频条保留全局对齐关系

---

## 6. 状态模型建议

建议新增以下核心状态：

- `currentGlobalTimeMs`
- `workspaceWindow = { startMs, endMs }`
- `videoOffsetMs`
- `videoDurationMs`
- `draftOffsetMs`（拖动视频条时的临时预览值）

关键派生值：

- `workspaceDurationMs = workspaceEndMs - workspaceStartMs`
- `workspaceLocalTimeMs = currentGlobalTimeMs - workspaceStartMs`
- `videoLocalTimeMs = currentGlobalTimeMs - effectiveOffsetMs`
- `effectiveOffsetMs = draftOffsetMs ?? videoOffsetMs`

约束规则：

- `currentGlobalTimeMs` 始终限制在 `workspaceWindow` 内进行播放
- 视频显示时间由全局时间和 `offsetMs` 推导
- 图表和事件列表只过滤 `workspaceWindow` 范围内的数据

---

## 7. 首版实现建议

第一阶段：

- 保留现有播放逻辑
- 为 Timeline 增加工作区窗口状态
- 将单轨改为三轨显示

第二阶段：

- 接入视频条的全局位置渲染
- 支持拖动视频条修改 `offsetMs`
- 拖动时实时预览，松手后提交保存

第三阶段：

- 图表、事件、地图高亮统一切换到工作区过滤
- 将现有 `Sync` 弹窗保留为精调模式

---

## 8. 一句话定义

这次升级不是简单地“多加一条进度条”，而是把 Replay Workspace 的时间系统升级为：

- 用总时间轴保留全局上下文
- 用视频时间轴完成全局对齐
- 用工作区时间轴承载局部聚焦和实际操作

这会让用户更自然地按片段复盘，而不是在整条长时间轴上反复粗略拖动。
