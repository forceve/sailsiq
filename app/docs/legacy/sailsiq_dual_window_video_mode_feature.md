# SailSIQ 双窗口视频辅助模式功能文档（增量版）

> 本文档只描述“新增实现功能”。
> 现有 Replay、视频导入、视频对齐、Overlay / Split、时间轴、地图、图表、HUD 等既有能力不在本文展开。

---

## 1. 文档范围

### 1.1 本文只覆盖
- 在现有 Replay Workspace 基础上新增“视频辅助窗”能力
- 双浏览器窗口之间的角色划分与同步规则
- 双窗口模式下新增的交互、状态、异常提示与边界

### 1.2 本文不覆盖
- 既有单窗口视频模式能力
- 既有视频导入、视频偏移校准、时间轴控制、地图与图表功能
- 未来可能扩展的多屏自动摆窗、高级权限控制、协同讲解

---

## 2. 背景与目标

现有 `Split View` 已支持在单个页面内同时显示视频与数据，但在“小屏设备 + 多窗口”场景下仍有两个问题：

- 单窗分屏后，视频与数据区域都被压缩，讲解体验下降
- 用户在双屏或多小屏环境下，无法把视频与数据分别放到两个独立窗口中使用

本次新增功能目标是：

- 让用户在保持现有 Replay 逻辑不变的前提下，额外打开一个独立的视频辅助窗
- 让主窗口专注数据与控制，辅助窗口专注视频展示
- 让两个窗口在同一 Session 下保持时间同步、播放同步、倍速同步

---

## 3. 功能定位

双窗口视频辅助模式不是新的 Replay 页面，也不是新的视频模式。

它是对现有视频辅助能力的增量扩展：

- 主窗口仍然是 Replay Workspace，承担数据展示与播放控制
- 新增一个“视频辅助窗”，只负责视频展示与跟随主窗口同步
- 两个窗口共同服务于同一个 Session 的讲解过程

---

## 4. 新增功能清单

### 4.1 打开视频辅助窗

在现有 Replay Workspace 中，新增“打开视频辅助窗”入口。

用户点击后：

- 系统打开一个新的浏览器窗口
- 新窗口绑定当前 Session
- 新窗口进入“视频辅助窗”角色
- 若辅助窗已存在，则优先激活已有窗口，而不是重复新开

### 4.2 窗口角色拆分

双窗口模式下新增两个明确角色：

- 数据主窗口
  - 保留现有 Replay 主工作流
  - 作为播放控制源
  - 负责时间轴、播放/暂停、跳转、倍速、视频偏移调整
- 视频辅助窗
  - 只显示视频相关内容
  - 跟随主窗口同步，不承担主控制职责
  - 用于外接第二块屏幕、投屏窗口或小屏设备独立展示

### 4.3 双窗口同步

新增跨窗口同步能力。主窗口状态变化时，辅助窗需要同步更新以下内容：

- 当前 Session 时间
- 播放/暂停状态
- 播放速度
- 视频偏移量
- 当前关联视频信息
- 视频模式上下文
- 当前是否处于 Sync 校准态
- 当前 Sync 校准预览所使用的视频时间点

同步原则：

- 主窗口为单一可信源
- 辅助窗默认被动跟随
- 跳转、拖动、快进、倍速修改后，辅助窗应在可接受延迟内跟上

### 4.4 双窗口下的 Sync 校准联动

在双窗口模式下，新增 Sync 校准过程中的实时联动预览。

当主窗口进入视频 `Sync` 操作时：

- 辅助窗进入“校准跟随态”
- 辅助窗的视频跟随主窗口当前的 Sync 预览时间，而不是只跟随主播放时间
- 用户在主界面拉动 Sync 进度条中的视频部分时，辅助窗视频画面需要实时变化
- 该联动以“实时更新画面” 为要求，即使当前处于暂停校准态，也要持续响应拖动位置

校准阶段的行为要求：

- 拖动过程中，辅助窗应连续更新到对应画面
- 不要求辅助窗在校准拖动时进入正常播放态
- 用户确认保存 offset 后，辅助窗切回按最新 offset 的常规同步状态
- 用户取消 Sync 后，辅助窗退出校准跟随态，恢复到原本播放上下文

### 4.5 辅助窗自动进入简化展示

视频辅助窗新增“简化显示规则”：

- 默认隐藏与数据工作流无关的侧边栏、图表区、标注列表、导出入口
- 默认保留视频画面与必要 HUD
- 默认不展示重复的主控制区，避免双窗口控制冲突

### 4.6 辅助窗连接状态反馈

新增双窗口连接状态提示。

至少包含以下状态：

- 已连接
- 连接中
- 主窗口已断开
- 会话不匹配
- 当前无可播放视频

当辅助窗失去同步来源时，用户应明确知道当前窗口不再跟随主窗口。

### 4.7 辅助窗生命周期管理

新增双窗口生命周期处理：

- 主窗口打开辅助窗后，记录当前辅助窗实例
- 辅助窗被手动关闭后，主窗口状态恢复为“未打开”
- 主窗口刷新或路由重进后，可重新连接辅助窗
- 同一 Session 内重复点击“打开视频辅助窗”时，不产生多个同类窗口

### 4.8 双窗口下的视频源切换响应

当主窗口发生以下变化时，辅助窗需要同步响应：

- 切换关联视频 URL
- 删除当前视频
- 修改视频类型
- 调整视频 offset

预期行为：

- 视频源变化后，辅助窗自动切换到最新视频
- 当前视频被移除后，辅助窗展示“无视频可播放”状态
- offset 更新后，辅助窗继续按新的同步关系播放

## 5. 新增交互流程

### 5.1 进入双窗口模式

1. 用户在 Replay Workspace 中进入已有视频的 Session
2. 用户点击“打开视频辅助窗”
3. 系统打开辅助窗并完成会话绑定
4. 主窗口显示“辅助窗已连接”
5. 用户继续在主窗口操作时间轴与播放控制
6. 辅助窗同步显示视频内容

### 5.2 辅助窗关闭后恢复

1. 用户手动关闭辅助窗
2. 主窗口识别到辅助窗断开
3. 主窗口更新状态为“辅助窗未连接”
4. 用户可再次点击入口重新打开

### 5.3 无视频场景

1. 用户未关联任何视频时点击入口
2. 系统不进入双窗口模式
3. 主窗口提示“当前 Session 无可播放视频”

### 5.4 浏览器阻止弹窗

1. 用户点击入口
2. 浏览器阻止新窗口打开
3. 主窗口提示“浏览器已阻止视频辅助窗，请允许弹窗后重试”

### 5.5 辅助窗失去同步

1. 辅助窗已打开
2. 主窗口关闭、刷新异常或通信断开
3. 辅助窗显示“主窗口已断开”状态
4. 辅助窗不再假装处于实时同步中

### 5.6 双窗口下的 Sync 校准预览

1. 用户已打开视频辅助窗
2. 用户在主窗口打开 `Sync` 校准
3. 用户拖动主界面 Sync 区域中的视频进度条
4. 辅助窗视频画面实时跟随变化
5. 用户保存 offset 后，辅助窗回到常规同步状态
6. 用户取消校准后，辅助窗退出校准跟随态

---

## 6. 新增状态定义

### 6.1 主窗口状态

- `companionClosed`：未打开辅助窗
- `companionOpening`：正在尝试打开辅助窗
- `companionConnected`：辅助窗已建立同步
- `companionBlocked`：浏览器阻止弹窗
- `companionDisconnected`：辅助窗曾连接但已断开

### 6.2 辅助窗状态

- `waitingForHost`：等待主窗口建立同步
- `connected`：已连接主窗口
- `hostDisconnected`：主窗口已断开
- `sessionMismatch`：会话不一致
- `noVideo`：当前无可播放视频
- `mediaBlocked`：浏览器阻止自动播放
- `mediaError`：视频不可播放

---

## 7. 本期范围约束

### 7.1 本期必须支持

- 一个数据主窗口 + 一个视频辅助窗
- 同一 Session 下的播放同步
- 打开、关闭、重连、断开提示
- 视频源变化与 offset 变化的同步响应

### 7.2 本期明确不做

- 一个 Session 同时打开多个视频辅助窗
- 数据窗与视频窗双向抢控制
- 自动把窗口精确摆放到指定显示器
- 本地文件视频在双窗口下的完整无感接力
- 双窗口模式下新增导出、分享、批注协同能力

---

## 8. 体验要求

### 8.1 同步体验

- 主窗口播放时，辅助窗应稳定跟随
- 主窗口拖动时间轴后，辅助窗应快速收敛到正确时间点
- 倍速切换后，辅助窗不应长时间处于明显漂移状态
- 主窗口进行 `Sync` 校准并拖动视频进度条时，辅助窗画面应实时更新
- `Sync` 校准期间，辅助窗不应停留在旧帧直到用户松手后才跳变

### 8.2 使用成本

- 用户应只通过一个明确入口进入双窗口模式
- 用户不需要理解复杂的“主从关系配置”
- 辅助窗关闭后，主窗口应保持现有工作流可继续使用

### 8.3 异常可理解

- 弹窗被拦截时，要给出明确提示
- 自动播放失败时，要给出明确提示
- 主窗口断开后，要给出明确提示

---

## 9. 验收标准

- 在已有视频的 Session 中，用户可以成功打开一个独立的视频辅助窗
- 主窗口播放、暂停、拖动、快进、倍速变化后，辅助窗能正确跟随
- 主窗口修改视频 offset 后，辅助窗继续按新 offset 正常同步
- 在双窗口模式下，主窗口执行 `Sync` 校准时，拖动视频进度条可驱动辅助窗实时更新画面
- 主窗口切换或移除视频后，辅助窗正确更新状态
- 辅助窗关闭、主窗口刷新、弹窗被浏览器拦截时，系统都有明确可见反馈
- 在无视频 Session 中，不允许误进入双窗口模式

---

## 10. 后续扩展方向（不属于本期）

- 自动检测第二屏并建议将辅助窗移动到目标屏幕
- 支持“数据辅助窗”与“视频辅助窗”双角色切换
- 支持双窗口下的本地文件视频无缝共享
- 支持演示模式、投屏模式、讲解模式的专用窗口皮肤
